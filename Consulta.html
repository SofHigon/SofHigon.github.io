Evolución Sistemas Computacionales


Sistema Monolítico Imagen Video
Un sistema monolítico es un tipo de diseño de software donde todos los componentes están integrados en una sola unidad.
  • Ventajas: Unificación del código, facilidad de desarrollo inicial.
  • Desventajas: Escalabilidad limitada, mantenimiento complejo.
  • Ejemplos: Aplicaciones empresariales tradicionales.

Sistema NTFS Imagen Video
NTFS (New Technology File System) es un sistema de archivos desarrollado por Microsoft con características avanzadas de seguridad y soporte para archivos grandes.
  • Ventajas: Seguridad avanzada, recuperación de datos.
  • Desventajas: No es compatible con sistemas antiguos.
  • Ejemplos: Sistemas de almacenamiento en Windows.

Sistema Servidor Imagen Video
Un sistema servidor proporciona servicios a clientes en una red, permitiendo la gestión centralizada de recursos y datos.
  • Ventajas: Seguridad, escalabilidad.
  • Desventajas: Costos de mantenimiento elevados.
  • Ejemplos: Servidores web, servidores de bases de datos.

Sistema Distribuido Imagen Video
Un sistema distribuido es un conjunto de computadoras que trabajan juntas como un único sistema lógico, mejorando escalabilidad y tolerancia a fallos.
  • Ventajas: Transparencia, escalabilidad, tolerancia a fallos.
  • Desventajas: Mayor complejidad en la administración.
  • Ejemplos: Redes P2P, sistemas en clúster.

Sistema Multinivel Imagen Video
Un sistema multinivel divide sus funciones en capas jerárquicas para mejorar la modularidad y escalabilidad.
  • Ventajas: Modularidad, mejor gestión de recursos.
  • Desventajas: Complejidad en la comunicación entre niveles.
  • Ejemplos: Arquitecturas cliente-servidor.

Sistema en la Nube Imagen Video
Un sistema en la nube permite acceder a recursos de computación a través de internet.
  • Ventajas: Escalabilidad, accesibilidad remota.
  • Desventajas: Dependencia de internet, riesgos de seguridad.
  • Ejemplos: Servicios como AWS, Google Cloud, Azure.
